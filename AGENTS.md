# Cognitive Bias Atlas Project Notes

## Product goal

Мы делаем русскоязычный `atlas-first` продукт про когнитивные искажения:

- публичный лендинг
- встроенный на лендинге интерактивный атлас искажений
- Telegram-курс с последовательным прохождением уроков
- AI-переписку по каждому уроку
- платные квизы на сайте, которые открываются после покупки в Telegram Stars

Главная идея продукта:

- человек приходит на лендинг
- видит красивую карту искажений и цепляющий маркетинговый copy
- заходит в Telegram-бот
- проходит курс последовательно от начала до конца
- после уроков возвращается на сайт за практикой, картой и квизами

## Current product surfaces

### 1. Landing page

Лендинг не должен содержать технический copy.

На лендинге пишем только:

- для кого продукт
- какую проблему решает
- что человек получит
- почему формат удобный
- как выглядит путь пользователя
- почему стоит зайти в Telegram
- почему хочется купить квизы

На лендинге нельзя писать:

- архитектурные решения
- названия внутренних сервисов в пользовательских блоках
- “MVP”, “n8n”, “Supabase”, “OpenRouter”, “Docker” как часть public copy
- технические объяснения “как это работает под капотом”

Landing tone:

- сильный маркетинговый русский язык
- короткие понятные тезисы
- обещание результата, а не описание реализации
- можно использовать эмодзи, но уместно и умеренно

### 2. Atlas

Атлас должен быть `сразу на лендинге`, а не жить как основная отдельная страница.

Требования к атласу:

- радиальная структура по мотивам референса `Cognitive_biases_diagram_RU.png`
- в центре — центральный символ/brain core
- вокруг центра — большой круг
- внутри круга — 4 большие смысловые группы
- внутри групп — подгруппы и конкретные искажения
- подписи должны ориентироваться радиально, по направлению от центра к внешнему радиусу
- карточка искажения открывается по клику на сам элемент карты
- карточка не должна жить в постоянной правой колонке
- карта должна выглядеть как mind map / poster / encyclopedia diagram, а не как dashboard block

Большие группы в атласе:

1. `Когда запоминаем и вспоминаем`
2. `Когда много информации`
3. `Когда не хватает смысла`
4. `Когда быстро реагируем`

Цвета именно для атласа:

- мягкий синий
- мягкий зеленый
- мягкий желтый
- мягкий красный

Общий сайт при этом остается в темной оранжевой палитре.

### 3. Telegram bot

Курс в боте идет `последовательно`, а не как свободная коллекция карточек.

Под каждым уроком строго 4 кнопки:

1. `Следующий`
2. `Атлас`
3. `Сохранить`
4. `Спросить AI`

Важно:

- на карточках сайта не надо давать кнопку “урок в Telegram” для каждого отдельного искажения
- основная логика курса — последовательное прохождение
- Telegram deep links нужны в CTA продукта, но не как хаотичная навигация по каждой карточке

AI flow в Telegram:

- у каждого урока есть 3 готовые подсказки
- пользователь может задать свой вопрос текстом
- пользователь может задать свой вопрос голосом
- голос должен транскрибироваться
- AI отвечает только в контексте текущего урока
- short memory хранится на уровне `(user_tg_id, lesson_id)`

### 4. Quiz pack

Квизы живут на сайте.

Монетизация:

- бесплатный основной курс
- платный `quiz pack`
- покупка разовая, не подписка
- покупка совершается в Telegram через Stars
- доступ на сайте проверяется по `tg_id` через Telegram auth / mini app init data

## Visual language

### Site-level visual rules

- общий фон сайта темный
- основная палитра сайта — оранжевые оттенки и теплые темные нейтрали
- CTA-кнопки — оранжевые
- стеклянные/прозрачные панели допустимы, если не ломают читаемость
- шапка должна появляться при прокрутке вниз и фиксироваться сверху
- блоки должны иметь достаточные вертикальные отступы и воздух

### Typography

- copy должен ощущаться как premium editorial / thoughtful product
- заголовки — плотные, уверенные, без техничности
- описания — человеческие, без канцелярита
- нельзя оставлять временные заглушки и “случайные” тексты

## Architecture

### Public app

- `Next.js App Router`
- деплой сайта на `Vercel`
- мини-приложение регистрации также живет внутри Next.js
- route mini app регистрации: `/miniapp/register`
- лендинг, атлас, карточки и квизы живут в одном приложении

### Automation and bot backend

- `n8n` — центр Telegram workflow
- логика: onboarding, выдача уроков, callbacks, AI-ветка, платежи, entitlements
- локально n8n поднимается в Docker
- для Telegram webhook локально используем `ngrok`
- для mini app / публичного сайта локально нужен отдельный `ngrok` tunnel или Vercel preview

### Data layer

- `Supabase / Postgres`
- storage для картинок уроков и atlas assets
- все основные сущности хранятся в таблицах `cb_*`

### AI

- провайдер: `OpenRouter`
- модель на старте — бесплатная
- модель задается через env
- позже модель можно менять без переписывания UX

## Copy rules for future edits

- никакого placeholder-copy в public UI
- никакой техничности на лендинге
- тексты писать как marketing/product copywriter для digital education product
- формулировки должны отвечать на вопросы:
  - что это
  - зачем это человеку
