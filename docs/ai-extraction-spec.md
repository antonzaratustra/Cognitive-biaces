# AI Extraction Spec For Cognitive Bias Atlas

## Goal

Эта инструкция нужна для внешнего AI, который анализирует:

- книгу
- изображение атласа
- дополнительные заметки или таблицы

и должен вернуть **структурированные данные** по когнитивным искажениям в формате, пригодном для загрузки в нашу базу и использования в:

- сайте
- радиальном атласе
- Telegram-курсе
- AI-ответах внутри уроков

Задача не в том, чтобы пересказать книгу. Задача в том, чтобы извлечь **каноническую базу знаний** по 160+ искажениям в строго нормализованном формате.

## Product Context

Мы строим русскоязычный `atlas-first` продукт про когнитивные искажения.

Важно:

- атлас имеет 4 большие сферы
- внутри сфер есть подгруппы
- внутри подгрупп находятся отдельные искажения
- каждое искажение должно подходить сразу для:
  - карточки в атласе
  - урока в Telegram
  - AI-контекста по уроку

## Fixed Top-Level Spheres

Все искажения обязательно должны быть привязаны ровно к одной из 4 сфер:

1. `memory` — `Когда запоминаем и вспоминаем`
2. `information` — `Когда много информации`
3. `meaning` — `Когда не хватает смысла`
4. `reaction` — `Когда быстро реагируем`

Нельзя придумывать пятый верхний раздел.

## Required Output Layers

AI должен вернуть данные в 3 слоя:

1. `taxonomy`
2. `bias records`
3. `relations`

### 1. Taxonomy

Нужно извлечь:

- 4 fixed spheres
- список подгрупп внутри сфер

Каждая подгруппа должна иметь:

- `key`
- `sphere_key`
- `title_ru`
- `description_ru`

Пример:

```json
{
  "key": "memory_false_reconstruction",
  "sphere_key": "memory",
  "title_ru": "Искажения реконструкции памяти",
  "description_ru": "Когда память достраивает, переписывает или упрощает прошлое."
}
```

### 2. Bias Records

Каждое искажение должно быть отдельной структурированной записью.

### 3. Relations

Нужно извлечь связи:

- `sphere -> subgroup`
- `subgroup -> bias`
- `bias -> related_bias`

## Preferred Delivery Format

Лучший формат ответа:

1. `taxonomy.json`
2. `biases.jsonl`
3. `relations.jsonl`

### taxonomy.json

```json
{
  "spheres": [
    {
      "key": "memory",
      "title_ru": "Когда запоминаем и вспоминаем"
    }
  ],
  "subgroups": [
    {
      "key": "memory_false_reconstruction",
      "sphere_key": "memory",
      "title_ru": "Искажения реконструкции памяти",
      "description_ru": "Когда память достраивает прошлое."
    }
  ]
}
```

### biases.jsonl

Каждая строка — отдельный JSON-объект.

```json
{"slug":"confirmation-bias","title_ru":"Искажение подтверждения","sphere_key":"information","subgroup_key":"information_selective_attention"}
{"slug":"hindsight-bias","title_ru":"Эффект знания задним числом","sphere_key":"memory","subgroup_key":"memory_false_reconstruction"}
```

### relations.jsonl

```json
{"source_key":"information_selective_attention","target_key":"confirmation-bias","relation_type":"contains"}
{"source_key":"confirmation-bias","target_key":"belief-bias","relation_type":"related"}
```

## Mandatory Fields

Без этих полей запись нельзя считать готовой к импорту:

- `slug`
- `title_ru`
- `title_en`
- `sphere_key`
- `subgroup_key`
- `short_text`
- `full_text`
- `everyday_examples`
- `antidotes`
- `related_slugs`
- `atlas_blurb`
- `ai_context`

## Field Writing Rules

### General

- писать по-русски, кроме `title_en`, `slug` и технических ключей
- не писать канцеляритом
- не писать слишком академически
- не писать размыто
- не дублировать один и тот же смысл в каждом поле

### short_text

Должно быть:

- 1-2 предложения
- очень ясно
- без воды
- объясняет суть искажения

### full_text

Должно быть:

- 2-4 абзаца
- человеческий русский язык
- объяснение механики и последствий
- пригодно и для сайта, и для Telegram-урока

### everyday_examples

Нужно:

- 2-4 бытовых примера
- конкретные жизненные сцены
- не абстракции

### antidotes

Нужно:

- 2-4 практики
- короткие, применимые
- не философские советы

### self_check_questions

Нужно:

- 2 вопроса
- они должны помогать поймать искажение у себя

### ai_context

Это отдельное поле для Telegram AI.

Оно должно:

- кратко объяснять, что именно AI должен подчеркивать
- перечислять полезные практики
- указывать, с какими соседними искажениями путаница наиболее вероятна
- быть компактным, но содержательным

## Slug Rules

`slug` должен быть:

- на английском
- в lowercase
- через дефис
- уникальным
- стабильным

Примеры:

- `confirmation-bias`
- `hindsight-bias`
- `availability-heuristic`
- `fundamental-attribution-error`

Нельзя:

- использовать пробелы
- использовать кириллицу
- использовать случайные сокращения

## Subgroup Rules

Каждое искажение должно принадлежать:

- одной сфере
- одной основной подгруппе

Если искажение подходит сразу в несколько мест:

- выбрать основную группу
- остальные связи вынести в `related_slugs`

Не нужно дублировать одно и то же искажение в нескольких подгруппах.

## Source Quality Rules

Нужно:

- опираться на книгу и визуальную карту
- при необходимости сопоставлять с общеизвестным английским названием
- сохранять смысл, а не делать буквальный шумный пересказ

Нельзя:

- выдумывать новые искажения, которых нет в материале
- смешивать разные bias в одну запись
- терять canonical English name, если он общеизвестен

## Relation Rules

Для `related_slugs` выбирай только реально близкие и полезные связи:

- максимум 3-5 соседних искажений
- не делать список “всего похожего подряд”

Хорошие типы связей:

- похожий механизм
- соседняя когнитивная ошибка
- частая путаница
- логическое продолжение

## Recommended Extraction Workflow

Не извлекай все 160+ искажений одним ответом.

Работай батчами.

Рекомендуемый порядок:

1. Сначала построить `taxonomy`
2. Потом выгружать искажения по одной сфере
3. Потом проверить уникальность `slug`
4. Потом проверить, что у каждого bias есть `sphere_key` и `subgroup_key`
5. Потом собрать `relations`

Оптимальный размер батча:

- 15-25 искажений за раз

## Validation Checklist

Перед финальной отдачей нужно проверить:

- у всех записей уникальный `slug`
- у всех записей есть `title_ru` и `title_en`
- каждое искажение привязано к одной из 4 сфер
- каждое искажение привязано к одной подгруппе
- `short_text` не дублирует `full_text` слово в слово
- `everyday_examples` конкретные, а не абстрактные
- `antidotes` практичны
- `related_slugs` существуют в наборе данных
- `ai_context` написан как рабочая подсказка для AI, а не как маркетинговый текст

## Prompt Template For External AI

Ниже готовая инструкция, которую можно отдавать другому AI.

```md
Тебе нужно извлечь структурированные данные по когнитивным искажениям из книги и визуальной карты.

Не пересказывай книгу сплошным текстом.
Не пиши эссе.
Не используй markdown-таблицы.

Верни данные в строго структурированном формате для базы знаний.

Цели:
- построить taxonomy из 4 сфер и подгрупп
- извлечь отдельные bias records
- извлечь связи между близкими искажениями

4 фиксированные сферы:
1. memory — Когда запоминаем и вспоминаем
2. information — Когда много информации
3. meaning — Когда не хватает смысла
4. reaction — Когда быстро реагируем

Для каждого искажения верни JSON-объект со следующими полями:
- slug
- title_ru
- title_en
- sphere_key
- subgroup_key
- aliases_ru
- aliases_en
- short_text
- full_text
- mechanism
- why_it_matters
- everyday_examples
- work_examples
- signals
- antidotes
- self_check_questions
- related_slugs
- lesson_order
- difficulty
- atlas_blurb
- ai_context
- image_prompt
- source_refs
- confidence

Правила:
- русский язык для всех описательных полей
- английский только для slug, title_en и technical keys
- не выдумывай новые bias
- не смешивай два искажения в одну запись
- если одно искажение похоже на другое, укажи это в related_slugs
- если сомневаешься, понизь confidence

Сначала верни taxonomy.
После этого возвращай bias records батчами по 15-25 штук.

Формат ответа:

2. biases.jsonl

Строго без пояснительного текста вне JSON.
```

## Mapping To Our Database

Практическое соответствие таблицам:

### cb_sections

Сюда можно класть:

- 4 сферы
- подгруппы

### cb_lessons

Сюда идет основная сущность bias:

- `slug`
- `title_ru/title_en`
- `short_text`
- `full_text`
- `ai_context`
- `related_slugs`
- `lesson_order`
- `image_url` позже

### cb_atlas_nodes

Сюда идут:

- sphere nodes
- subgroup nodes
- bias nodes

### cb_atlas_edges

Сюда идут:

- `sphere -> subgroup`
- `subgroup -> bias`
- `bias -> bias`

## Final Principle

Нам нужны не просто “описания искажений”, а единый структурированный knowledge layer, который одинаково хорошо работает для:

- радиального атласа
- карточек на сайте
- Telegram-уроков
- AI-подсказок и ответов

Поэтому качество структуры важнее литературности.
